## Руководство по добавлению информации в Changelog

Когда вы делаете коммит, который влияет на функциональность приложения, добавьте запись в UNRELEASED секцию:

```markdown
## [Unreleased]

### Added
- Новая фча 1
- Новая фича 2

### Fixed
- Исправление бага 1

### Changed
- Изменение функции 1

### Removed
- Удаление старой функции
```

### Категории изменений:

- **Added** — новые функции
- **Changed** — изменения в существующей функциональности
- **Deprecated** — функции, которые скоро будут удалены
- **Removed** — удаленные функции
- **Fixed** — исправления ошибок
- **Security** — исправления уязвимостей

## Процесс релиза

1. Обновите версию в `package.json` всех рабочих пакетов (backend, frontend, print3d-app)
2. Переименуйте `[Unreleased]` на `[X.Y.Z] — YYYY-MM-DD`
3. Добавьте новый `[Unreleased]` раздел
4. Создайте git tag: `git tag -a vX.Y.Z -m "Release version X.Y.Z"`
5. Запушьте теги: `git push origin vX.Y.Z`
6. Обновите [RELEASE_INFO.md](./RELEASE_INFO.md)
