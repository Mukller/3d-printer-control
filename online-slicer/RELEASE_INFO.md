## Как обновиться

### С 0.0.1 на 0.1.0 (когда выпустится)

```bash
# 1. Резервная копия текущей версии
git checkout -b backup/v0.0.1

# 2. Обновить до последней версии
git checkout main
git pull origin main

# 3. Обновить зависимости
cd backend && npm update && cd ..
cd frontend && npm update && cd ..
cd print3d-app && npm update && cd ..

# 4. Проверить CHANGELOG для breaking changes
cat CHANGELOG.md

# 5. Перезапустить проект
cd backend && npm start &
cd ../frontend && npm run dev &
```

### Для Docker (планируется)

```bash
# Обновить образ
docker pull mukller/online-slicer:latest

# Перезапустить контейнер
docker-compose up -d --force-recreate
```

## Заметки по обновлению

### Breaking Changes (Несовместимые изменения)

При обновлении **всегда** проверяйте CHANGELOG на breaking changes:

```bash
# Перед обновлением
git log v0.0.1..origin/main --oneline
cat CHANGELOG.md
```

### Миграция БД

Если обновление включает изменения схемы БД:

```bash
# Создать резервную копию
mongodump --out backup/

# Выполнить миграцию
npm run migrate

# Проверить результаты
npm test
```

## Поддержка версий

### Long-Term Support (LTS)

- **v1.0.0+** — будут получать обновления безопасности в течение 2 лет
- **v0.x.x** — получают обновления в течение 6 месяцев после выпуска v1.0.0

### Жизненный цикл

```
Release → Active Support (3 месяца) → 
→ Maintenance (3 месяца) → End of Life
```
